# 最小代价方案行动计划（Agent 管理 + 轻量资产域）

## 目标与原则
- 目标：在不偏离“作业平台”核心的前提下，补齐最小可用的 Agent 管理与资产事实源，保障任务下发的可控性与安全性。
- 原则：小步快跑、分层解耦、RBAC 为主、作用域过滤为辅，不做重型 CMDB，对标“够用”而非“全量蓝鲸/codo”。

## 范围与不做的事
- 做：主机/Agent 基础模型、Agent token 发放/吊销、状态机与对账、作用域校验、最小审计与面板。
- 不做：复杂拓扑/业务树、全量 CMDB 能力、多云同步、灰度升级平台、复杂 ABAC/继承链。

## 关键交付物
- 资产域数据模型（主机、Agent、标签/业务/环境、状态机）。
- Agent token 管理（签发、吊销、绑定主机/Agent 记录）。
- 控制面 API 级作用域校验（业务/环境/标签过滤），作业下发前过滤目标。
- 最小面板与审计（在线/掉线/版本分布、错误码 TOP、告警；权限/令牌/Agent 操作日志）。
- 对账任务（期望清单 vs 实际在线 Agent）。

## 实施里程碑
1) 模型与常量（1-2 天）
   - 定义主机表、Agent 表、标签/业务/环境字段、状态机（未装/待激活/在线/掉线/禁用）。
   - 定义权限常量（资产域与作业域分开命名空间）。
2) Token 与注册流（1-2 天）
   - API：签发/吊销 agent token，绑定主机记录；注册/心跳落库 Agent 状态。
   - Agent/Agent-Server：沿用现有 token 流程，无需用户 RBAC。
3) 作用域校验落地（2 天）
   - 控制面 API：资产/Agent 接口增加 RBAC + 作用域（业务/环境/标签）过滤。
   - 作业下发：按作用域过滤目标主机/Agent，再生成任务分发列表。
4) 对账与告警（1-2 天）
   - 周期任务：对账“期望清单 vs 在线列表”，输出未装/掉线/重复/版本不一致。
   - 告警：掉线超时、版本过期、重复注册。
5) 面板与审计（2 天）
   - 面板：在线/掉线计数、版本分布、错误码 TOP。
   - 审计：权限变更、token 发放/吊销、Agent 安装/禁用、作业执行/取消。
6) 稳定与补坑（视情况）
   - 压测与容量估算；补文档、操作指南。

## 权限模型（简化版）
- 模式：RBAC + 作用域过滤（业务/环境/标签）；默认拒绝，白名单授权。
- 资产域动作示例：view_host/add_host/edit_host/delete_host/bind_agent/unbind_agent；view_agent/install_agent/upgrade_agent/disable_agent/revoke_token/view_logs/view_metrics。
- 作业域保持现状，仅在“目标筛选”前增加作用域校验。

## 运行时校验策略
- 控制面：强校验（用户权限 + 作用域过滤），确保下发目标在授权范围内。
- Agent-Server：仅校验 agent token 与绑定关系，不承担用户级鉴权。
- Agent：只做 token 注册/心跳/执行。

## 最小告警与对账
- 掉线：心跳超时。
- 版本：Agent 版本落后于期望版本。
- 重复：同主机/标签绑定的多实例在线。
- 对账：日/小时级任务生成报表。

## 交付优先级
- 必须：模型/状态机、token 管理、API 作用域校验、对账与掉线告警。
   - 价值：安全边界 + 基本可见性。
- 次优：版本/重复告警、错误码 TOP、简单面板。
- 可后置：多云同步、灰度升级、mTLS/签名、复杂 ABAC。

## 文档输出
- 本计划文件。
- 快速使用指引：如何签发 token、如何注册 Agent、如何查看/过滤目标并下发作业。
- 运维手册：对账与告警处理流程。

## “三板斧”对照（与里程碑顺序一致）
1) 轻量资产源：主机/节点表 + 标签 + 作用域（业务/环境/区域） + agent 接入配置/凭据索引，仅用于挑选目标和给 agent 提供配置，不做重型 CMDB。
2) 鉴权最小闭环：谁可以对哪类主机/标签执行作业/安装/升级；凭据 use 与 view 分离（执行可用、默认不可见）；prod 批量可二次确认或限流。
3) 审计与安全护栏：写操作/管控动作落审计；批量/高危操作有限流和环境隔离；监控先用现有日志/心跳与掉线/版本告警，后续再扩展。

对照关系：三板斧 = 本文“数据模型 + 作用域鉴权 + 审计/告警/护栏”，落地仍按里程碑 1-5 推进，先 dev/stage 验证，再扩 prod。
